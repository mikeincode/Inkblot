# Social Language

Language is one of the most important systems in Inkblot.

Most platforms inherit terminology from:
- social media
- productivity software
- forums
- corporate tools

These words shape user behavior unconsciously.

Inkblot intentionally rejects generic platform language in favor of terminology that reinforces:
- mythology
- participation
- interpretation
- historical identity
- collaborative worldbuilding

The goal is not aesthetic novelty alone.

The language itself helps define the emotional reality of the platform.

---

# Core Principle

Words create behavioral framing.

Calling something a:
- “post”
feels temporary and disposable.

Calling it a:
- “Node”
feels structural and historical.

Language changes how users perceive:
- contribution
- legitimacy
- participation
- authority
- creativity
- identity

Inkblot’s terminology is part of the system design itself.

---

# Why Generic Language Fails

Generic platform language encourages:
- shallow interaction
- content consumption
- disposable engagement
- algorithmic thinking
- social performance

Examples:
- likes
- posts
- followers
- comments
- feeds
- creators
- trending

These terms carry emotional assumptions from modern social media ecosystems.

Inkblot intentionally avoids this framing.

---

# Mythological Framing

Inkblot language should always reinforce the feeling that users are:
- participating in living mythology
- shaping evolving history
- contributing to civilizations
- entering interpretive ecosystems

rather than:
- publishing content into a feed

The emotional difference matters.

---

# Canonical Language Rules

When possible:
prefer mythology, archival, or ecosystem terminology over generic software terminology.

Examples:

| Avoid | Prefer |
|---|---|
| Post | Node |
| Like | Resonance |
| Comment | Voice |
| Group | Faction |
| Forum | Chamber |
| Moderator | Keeper |
| User | Contributor / Chronicler |
| Community | Order / Collective / Circle |
| Category | Domain / Branch |
| Thread | Discussion Weave / Chamber Thread |
| Admin | Keeper |
| Upload | Inscribe / Submit |
| Draft | Unformed Node |
| Publish | Release / Plant |
| Deleted | Archived |
| Trending | Rising |
| Notifications | Signals |
| Activity Feed | Living Record |

Not every replacement must be absolute.

Clarity still matters.

But the emotional framing should remain consistent.

---

# Resonance

“Resonance” replaces likes or upvotes.

The term was chosen because:
- mythology spreads through emotional resonance
- cultural ideas survive through resonance
- interpretation gains power through resonance

Resonance implies:
- alignment
- impact
- narrative gravity
- cultural influence

not shallow approval.

---

# Voices

Replies and discussion contributions are called Voices.

This reinforces the idea that:
mythology evolves through many speaking perspectives.

A Voice is not just a comment.

It is participation in collective interpretation.

---

# Chambers

Discussion spaces are called Chambers.

The term was chosen because it evokes:
- hidden halls
- lore archives
- underground councils
- interpretive gatherings
- mythological debate spaces

A Chamber should feel:
localized,
ritualistic,
historical.

Not like a generic forum thread.

---

# Factions

Groups are called Factions.

This changes the emotional expectation from:
“social grouping”

to:
“ideological interpretation order.”

Factions are meant to feel:
- philosophical
- political
- mythological
- historically motivated

not casual clubs.

---

# Keepers

Moderators and canon authorities are called Keepers.

The term implies:
- stewardship
- preservation
- archival responsibility
- mythological guardianship

rather than platform policing.

---

# Nodes

Content units are called Nodes.

The term reinforces:
- interconnected mythology
- structural contribution
- ecosystem growth
- narrative architecture

Nodes feel:
persistent,
historical,
structural.

Not disposable.

---

# Tree & Shadow Language

The contrast between Tree and Shadow language is intentional.

## Tree Language
Should feel:
- formal
- archival
- historical
- sacred
- preserved

Examples:
- Canonized
- Keeper Reviewed
- Rooted
- Inscribed
- Official Continuity

---

## Shadow Language
Should feel:
- unstable
- underground
- emergent
- folkloric
- partially uncontrollable

Examples:
- Mutation
- Divergence
- Rumor
- Consensus
- Unbound
- Forked
- Rising

The linguistic contrast helps reinforce ecosystem identity.

---

# Avoiding Corporate Tone

Inkblot language should never sound:
- productivity-oriented
- startup-branded
- hyper-optimized
- algorithmic
- engagement-maximized

Avoid:
- growth hacking language
- creator economy terminology
- corporate community language
- sterile UX copy

The platform should sound:
human,
mythological,
historical,
alive.

---

# Interface Writing Style

UI writing should generally feel:
- literary
- atmospheric
- restrained
- emotionally intentional

Avoid:
- excessive cheerfulness
- gamified excitement
- meme-heavy language
- corporate friendliness

Inkblot should feel:
quietly mythic.

---

# Emotional Weight

Actions should feel meaningful through language.

Examples:

Instead of:
“Your post was approved.”

Prefer:
“Your Node has been canonized into the Trunk.”

---

Instead of:
“You joined a group.”

Prefer:
“You have pledged alignment to this Order.”

---

Instead of:
“Comment added.”

Prefer:
“Your Voice now echoes within the Chamber.”

The goal is not roleplay.

The goal is emotional framing.

---

# Language Consistency

Consistency matters.

If Inkblot mixes:
- mythological terminology
with
- generic SaaS language

the emotional illusion collapses.

The platform’s identity depends heavily on linguistic cohesion.

---

# Final Principle

Inkblot should sound like a civilization speaking about its own mythology.
