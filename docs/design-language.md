# Design Language

Inkblot is not designed to feel like modern social software.

The platform should feel like:
- living mythology
- archival history
- forbidden lore
- evolving civilizations
- interpretive archaeology
- mythological ecosystems

Every visual, interaction, animation, and word choice should reinforce this emotional identity.

---

# Core Emotional Tone

Inkblot should feel:

- cinematic
- atmospheric
- ancient
- alive
- mysterious
- archival
- organic
- evolving
- emotionally heavy
- culturally layered

The platform should evoke the feeling of:
- discovering lost histories
- entering hidden libraries
- mapping mythological lineages
- participating in living folklore
- witnessing civilizations evolve through narrative

---

# What Inkblot Must Never Feel Like

Inkblot must never resemble:
- generic productivity software
- corporate SaaS dashboards
- modern social media feeds
- startup minimalism
- sterile interfaces
- influencer platforms
- content farms
- engagement-maximizing systems

Avoid:
- excessive white space
- flat sterile design
- bright corporate palettes
- shallow gamification
- generic social UI patterns

Inkblot is mythology infrastructure.
Not a productivity tool.

---

# Visual Philosophy

The interface should feel:
like an evolving archive rather than an application.

UI elements should resemble:
- illuminated manuscripts
- archival records
- living diagrams
- branching maps
- forgotten machinery
- cosmic cartography
- ancient systems growing in darkness

The world itself should appear to breathe and evolve.

---

# Color Philosophy

## Tree Palette
Represents:
- canon
- governance
- continuity
- preservation
- historical structure

Primary emotional colors:
- indigo
- deep blue
- muted emerald
- silver
- dim archival gold

The Tree should feel:
stable, sacred, rooted, historical.

---

## Shadow Palette
Represents:
- mutation
- reinterpretation
- divergence
- underground mythology
- emergent folklore

Primary emotional colors:
- violet
- dark magenta
- bruised purple
- blackened crimson
- ghostly gradients

The Shadow should feel:
alive, unstable, seductive, mythologically dangerous.

---

# Lighting Philosophy

Inkblot should primarily exist in darkness.

Light should feel:
- earned
- localized
- atmospheric
- symbolic

Use:
- glow
- soft bloom
- pulse effects
- ambient gradients
- drifting particles
- subtle fog
- low-opacity silhouettes

Avoid:
- harsh brightness
- flat illumination
- sterile clarity

Darkness gives mythology weight.

---

# Typography

## Serif Typography
Primary narrative voice.

Use for:
- universe names
- lore
- mythology text
- ceremonial headings
- archival sections

Typography should feel:
- literary
- timeless
- mythological
- emotionally heavy

Cormorant Garamond became foundational because it feels:
elegant, historical, and slightly haunted.

---

## Sans Typography
Used sparingly for:
- system labels
- metadata
- navigation
- interface utility

Sans text should remain subdued and secondary to narrative typography.

---

# Motion Language

Animation should feel:
- organic
- ceremonial
- atmospheric
- slow
- alive

Preferred motion:
- branch growth
- drifting particles
- breathing glow
- ink spread
- root expansion
- pulse rhythms
- timeline emergence
- scroll-triggered revelation

Avoid:
- aggressive bounce
- hyperactive UI
- mobile-app energy
- attention-grabbing spam motion

Motion should create:
mythological atmosphere,
not stimulation addiction.

---

# Structural Metaphors

Inkblot’s UI should consistently reinforce:
living mythology systems.

Common metaphors:
- Trees
- Roots
- Branches
- Veins
- Ink
- Archives
- Lineage
- Ritual
- Constellations
- Manuscripts
- Sigils
- Cartography

The interface itself should feel partially alive.

---

# Social Interface Philosophy

Inkblot avoids generic social mechanics.

Examples:

| Generic Platform | Inkblot |
|---|---|
| Likes | Resonance |
| Comments | Voices |
| Groups | Factions |
| Forums | Chambers |
| Moderators | Keepers |
| Posts | Nodes |

Language shapes emotional behavior.

Users should feel like:
participants in mythology,
not consumers in a feed.

---

# Information Density

Inkblot should balance:
- readability
- mystery
- atmospheric layering

Information should feel:
discovered,
not dumped.

The interface should encourage:
exploration,
curiosity,
interpretation.

---

# Spatial Feeling

The UI should feel:
deep.

Not flat.

Use:
- layered gradients
- ambient silhouettes
- distant visual structures
- parallax suggestion
- fading depth
- soft environmental movement

Worlds should appear larger than the visible screen.

---

# Narrative Weight

Every action should feel consequential.

Examples:
- Canonization should feel ceremonial
- Branch creation should feel evolutionary
- Joining a Faction should feel ideological
- Entering Shadow should feel dangerous
- Planting a Universe should feel mythic

Interactions should carry emotional gravity.

---

# The Tree & Shadow Contrast

The relationship between Tree and Shadow should be visually obvious.

## The Tree
Feels:
- ordered
- luminous
- rooted
- structured
- preserved

## The Shadow
Feels:
- unstable
- spreading
- seductive
- partially uncontrollable
- culturally alive

This contrast is one of the central emotional systems of Inkblot.

---

# Design Rule

If a design decision makes Inkblot feel more like:
- LinkedIn
- Reddit
- Discord
- Jira
- Notion
- generic SaaS

then it is probably wrong.

If it makes Inkblot feel more like:
- a living archive
- a mythological machine
- an evolving civilization
- a forbidden library
- narrative archaeology

then it is probably correct.

---

# Final Principle

Inkblot should always feel like a world remembering itself.
