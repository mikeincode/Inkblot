# Inkblot MVP

This document defines the minimum viable version of Inkblot.

The goal of the MVP is NOT to build every imagined system immediately.

The goal is to prove the core experience:

> Can people collaboratively grow living mythology in a way that feels meaningful, atmospheric, and structurally different from existing platforms?

Everything in the MVP should support answering that question.

---

# Core MVP Philosophy

The MVP should prioritize:

- emotional identity
- mythology participation
- collaborative storytelling
- Tree/Shadow dynamics
- contribution workflows
- ecosystem clarity

The MVP should NOT prioritize:
- scale
- monetization
- algorithmic systems
- advanced optimization
- massive feature breadth

Inkblot succeeds or fails based on:
whether the mythology ecosystem feels alive.

---

# MVP Experience Goal

A new user should be able to:

1. Enter Inkblot
2. Discover a Universe
3. Understand the Tree vs Shadow concept
4. Explore Branches and Timelines
5. Read Nodes
6. Enter a Chamber
7. Contribute a Node
8. Experience canon review or Shadow release
9. Feel emotionally connected to the evolving mythology

If this loop feels compelling,
the foundation works.

# MVP Technical Priorities

The current prototype is heavily visual.

The next stage of the MVP is transitioning from:
visual simulation
to
real application infrastructure.

Priority order matters.

---

# Infrastructure Priorities

## Phase 1 — Core Foundation

### Authentication
Required:
- signup/login
- sessions
- identity persistence

Suggested:
- Clerk
- Auth.js
- Supabase Auth

---

### Database Integration
Required:
- real Universes
- real Nodes
- real Branches
- real users
- real Chambers

Current visual data must become persistent.

Suggested:
- PostgreSQL
- Drizzle ORM

---

### API Layer
Required:
- CRUD endpoints
- authentication validation
- contribution workflows
- canonization flows

Current architecture already supports this direction.

---

# Phase 2 — Functional Ecosystem

## Real Contribution Flow
Users must be able to:
- submit Nodes
- create discussions
- join factions
- interact with Chambers
- contribute to universes

with actual persistence.

---

## Real Tree Relationships
The Tree structure must become:
- relational
- queryable
- expandable

Branches and Timelines should function as real graph structures.

---

## Real Shadow Mechanics
The Shadow should support:
- unofficial branches
- mutation chains
- consensus growth
- branch popularity
- cultural divergence

---

# Phase 3 — Discovery & Retention

## Discovery System
Users need ways to discover:
- universes
- rising branches
- active Chambers
- influential factions
- mythology conflicts

---

## Notifications / Signals
Users should receive Signals for:
- canonization
- replies
- faction activity
- branch developments
- lineage events

---

## Search
Critical long-term feature.

Search should eventually support:
- Nodes
- Universes
- Timelines
- factions
- contributors
- mutations

---

# Features Explicitly NOT Required For MVP

Avoid premature complexity.

Not required initially:
- mobile app
- monetization
- AI generation systems
- realtime multiplayer editing
- advanced recommendation algorithms
- marketplace systems
- blockchain/NFT systems
- complex moderation tooling
- video/audio systems

The mythology loop matters more than scale features.

---

# MVP Success Metrics

The MVP succeeds if users:
- emotionally understand the ecosystem
- create meaningful Nodes
- debate canon
- form interpretations
- grow branches organically
- revisit universes voluntarily

The key metric is:
mythology participation.

Not raw engagement numbers.

---

# Biggest MVP Risk

The largest risk is:
building atmosphere without real contribution gravity.

Inkblot cannot survive as:
“beautiful fake UI.”

The systems must eventually become real enough for:
- genuine collaboration
- real governance
- emotional investment
- persistent mythology history

The transition from prototype → living ecosystem is the critical phase.

---

# MVP Identity Rule

If forced to choose between:
- more features
or
- stronger mythology feeling

prioritize mythology feeling.

That is the actual product.

---

# Final Principle

The MVP does not need to prove scale.

It only needs to prove that worlds can feel alive.
